# TripPilot 실제 사용 시나리오 기반 수동 QA 체크리스트

아래 체크리스트는 개발자가 아닌 실제 테스터가 앱을 직접 조작하며 검증할 수 있도록 작성했다.

---

## QA-01
- **ID**: QA-01
- **영역**: 계정/로그인
- **사전 조건**: 테스트 계정(일반 사용자) 1개, 이메일 인증 가능 상태
- **테스트 절차**:
  1. 앱 실행 후 로그인 화면 진입
  2. 정상 계정으로 로그인
  3. 로그아웃 후 잘못된 비밀번호로 로그인 시도
- **기대 결과**:
  - 정상 로그인 성공 후 홈 화면으로 이동
  - 로그아웃 시 세션 삭제 및 로그인 화면 복귀
  - 잘못된 비밀번호는 명확한 오류 문구 표시
- **실패 시 확인할 로그/DB/API**: Auth API 응답 코드, Supabase auth 로그, 클라이언트 에러 로그
- **심각도**: 높음
- **release blocker 여부**: 예

## QA-02
- **ID**: QA-02
- **영역**: 온보딩
- **사전 조건**: 신규 계정(온보딩 미완료)
- **테스트 절차**:
  1. 신규 계정으로 최초 로그인
  2. 온보딩 단계(관심사/여행 스타일/예산) 입력
  3. 완료 후 메인 탭 이동 확인
- **기대 결과**:
  - 단계별 입력 저장
  - 완료 후 온보딩 재노출 없음
- **실패 시 확인할 로그/DB/API**: profile/onboarding 저장 API, 사용자 프로필 테이블
- **심각도**: 중간
- **release blocker 여부**: 아니오
