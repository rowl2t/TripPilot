# 보안 정책

## 신고 방법
보안 이슈는 공개 이슈 대신 비공개 채널로 신고해 주세요.

## 기본 원칙
- 최소 권한 원칙
- 민감정보(키/토큰) 저장 금지
- 서버 사이드 비밀키는 클라이언트 번들에 포함 금지
- 로그에 PII/토큰 기록 금지

## 점검 항목
- 인증/인가 우회 가능성
- SQL/RLS 정책 누락
- 외부 링크/리다이렉트 오용
- 의존성 취약점

## 릴리즈 전 확인
- `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm build`
- 환경변수 분리(dev/staging/prod)
- mock provider 비활성화 여부 확인
